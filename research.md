# Глубокое исследование ниши — ИН.ЯЗ
> Онлайн-школа английского языка для детей 10–17 лет. Дата: май 2026.

---

## Раздел D. Экспертная оценка — лендинг-портфолио цифрового мастера

> Проект: лендинг-портфолио специалиста, создающего сайты, приложения и ИИ-агентов.
> Оценка с трёх точек зрения: UX-дизайнер, маркетолог по конверсии, владелец бизнеса.

---

### Эксперт 1 — UX-дизайнер (10 лет опыта)

#### Типичные ошибки новичков в структуре

**1. «Обо мне» вместо «Что я сделаю для вас»**
Самая частая ошибка — герой-секция начинается с рассказа о себе: «Меня зовут Иван, я фулстек-разработчик с 5-летним опытом». Посетителя это не касается. Он пришёл с вопросом «можешь ли ты решить мою проблему?» — и сразу должен получить ответ «да, вот как».

**2. Портфолио без контекста**
Скриншоты сайтов без объяснения: что за проект, какую задачу решал, какой результат дал. Посетитель смотрит на красивые картинки, но не понимает, что за ними стоит. Кейс без цифр — не кейс.

**3. Перегруженная навигация**
Меню с 8 пунктами на лендинге. Лендинг — это не многостраничный сайт. Максимум 4–5 якорных ссылок, и то не обязательно. Цель одна — заявка.

**4. Форма с 7 полями**
«Имя, фамилия, телефон, email, тип проекта, бюджет, срок, опишите задачу» — при виде этого 80% закрывают страницу. Первый контакт: имя + один канал связи. Всё остальное — на созвоне.

**5. Нет мобильной версии или она сломана**
Портфолио разработчика с плохой мобильной версией — это приговор. Клиент думает: «Если он не может сделать нормальную мобилку себе — что он сделает мне?»

**6. Скорость загрузки убита тяжёлыми изображениями**
Портфолио с 15 полноразмерными скриншотами по 3 МБ каждый. Страница грузится 8 секунд. Конверсия падает на 4% за каждую лишнюю секунду загрузки.

---

#### Что обязательно должно быть

| Элемент | Почему |
|---|---|
| Чёткий заголовок — результат для клиента | Первые 3 секунды решают всё |
| 2–4 кейса с описанием задачи и результата | Доказательство умений в контексте |
| Конкретный список услуг | Клиент должен понять: «это — про меня» |
| Один CTA, повторённый 3 раза по странице | Не заставлять искать, куда нажать |
| Форма с 2 полями максимум | Снизить барьер первого контакта |
| Время ответа / следующий шаг | Снять тревогу «что будет после заявки» |
| Социальное доказательство | Отзыв, логотип клиента или конкретная цифра |

---

#### Что раздражает пользователей и заставляет уйти

- **Автоплей видео со звуком** — закрывают моментально
- **Всплывающее окно через 3 секунды** — «Подпишитесь на рассылку!» на лендинге разработчика — это нонсенс
- **Анимации, которые мешают читать** — текст появляется по одной букве, карточки дёргаются — красиво, но раздражает
- **Нет цен даже ориентировочно** — «стоимость по запросу» = «скорее всего дорого», уходят
- **Нечитаемый шрифт на фоне** — белый тонкий текст на светло-сером градиенте
- **Не понятно, с кем я говорю** — нет фото, нет имени, ноль личности. «Студия» из одного человека, но притворяющаяся корпорацией — не работает в нише фриланса
- **Бесконечный скролл без структуры** — посетитель не понимает, где находится на странице

### Эксперт 2 — Маркетолог по конверсии

#### Какой оффер зацепит клиента

Слабый оффер (встречается у 90% разработчиков):
> «Создаю сайты, приложения и ИИ-агентов под ключ»

Это описание деятельности. Оно не цепляет, потому что так пишут все.

Сильный оффер — формула: **[результат] для [кто] за [срок/условие]**

Примеры:
> «Запускаю сайт для малого бизнеса за 7 дней — с оплатой только после приёмки»

> «Автоматизирую рутину вашего бизнеса ИИ-агентом — вы экономите 20+ часов в месяц»

> «Делаю лендинги, которые продают, а не просто висят в интернете»

Ключ: говорить языком результата клиента, а не языком своего стека технологий. Клиенту не важен React — ему важно, что сайт будет быстрым и даст заявки.

---

#### Триггеры доверия — что убеждает платить незнакомому человеку

**Уровень 1 — базовый (без него никак):**
- Реальное фото и имя — не логотип «студии», а живой человек
- 2–3 отзыва с именем, должностью, компанией клиента и конкретным результатом
- Ссылки на живые проекты (не просто скриншоты — кликабельные URL)

**Уровень 2 — усиленный:**
- Логотипы компаний-клиентов (даже 3–5 небольших — уже работает)
- Конкретные цифры: «17 проектов сдано», «средний срок 12 дней», «клиенты из 4 стран»
- Гарантия или условие снижения риска: «правки до вашего ОК», «предоплата 30%»

**Уровень 3 — конкурентное преимущество:**
- Упоминание в СМИ, подкастах, профессиональных сообществах
- Открытый GitHub с активностью — технические клиенты проверяют
- Сертификаты (особенно для ИИ-направления: OpenAI, Google Cloud, AWS)

---

#### Путь посетителя от «зашёл» до «оставил заявку»

```
1. Попал на страницу (реклама / рекомендация / поиск)
         ↓
2. Первый экран — 3 секунды: «Это про меня?»
   → Да: читает дальше
   → Нет/непонятно: уходит
         ↓
3. Смотрит кейсы: «Он умеет делать то, что мне нужно?»
   → Есть похожий проект: интерес растёт
   → Нет похожего: сомнение
         ↓
4. Ищет подтверждение: отзывы, цифры, живые ссылки
   → Находит: доверие
   → Не находит: сомнение + уход
         ↓
5. Смотрит на цену / условия: «Это в моём бюджете?»
   → Понятно и приемлемо: готов к контакту
   → Непонятно: «позвоню потом» (= никогда)
         ↓
6. Видит CTA: «Что будет после нажатия?»
   → Понятный следующий шаг (созвон / бриф / ответ за 2 часа): жмёт
   → Непонятно что будет: не жмёт
         ↓
7. Форма — 2 поля: оставляет заявку ✓
```

**Где теряется больше всего людей:** шаги 2 и 5. Непонятный первый экран + отсутствие цен = 60–70% уходят не дойдя до портфолио.

---

#### Что усиливает конверсию на портфолио разработчика

- **«Бесплатная консультация 30 мин»** — снижает барьер первого контакта. Клиент ничем не рискует
- **Пример мини-проекта «для знакомства»** — «Сделаю лендинг за 3 дня за фиксированную сумму» — понятный вход
- **Дедлайн или ограничение** — «Беру 2 проекта в месяц. Ближайший слот — июнь» — создаёт ощущение дефицита без обмана, если это правда
- **Чёткий следующий шаг** — «Нажмите → я напишу в Telegram в течение часа → 30-минутный созвон → оценка проекта»

---

### Эксперт 3 — Владелец бизнеса (потенциальный клиент)

> Типичный профиль: малый или средний бизнес, не технарь, уже обжигался на фрилансерах, ценит своё время, принимает решения быстро.

#### Что он видит в первые 5 секунд

Он не читает — он сканирует. Взгляд проходит по странице за 3–5 секунд по F-паттерну: заголовок, первые слова каждого блока, изображения, кнопка.

**Что он ищет:**
1. «Это вообще про мой тип задачи?» — заголовок
2. «Кто этот человек?» — фото + имя
3. «Что-то похожее на моё уже делал?» — портфолио
4. «Сколько стоит?» — цена или хотя бы диапазон

Если за 5 секунд не нашёл ответ хотя бы на первые два — закрыл.

---

#### Что заставит остаться

- **Знакомый контекст** — видит в портфолио проект из своей или похожей ниши. «О, делал для ресторана — значит поймёт мой бизнес»
- **Конкретные слова о его боли** — «Устал от разработчиков, которые пропадают на полпути?» — если это его ситуация, он остановится
- **Человек, не корпорация** — фото, живой текст от первого лица, без канцелярита. Покупает не у «студии», а у человека, которому доверяет
- **Понятная цена или диапазон** — «от 30 000 ₽» или «типовой лендинг — 25 000–50 000 ₽» лучше, чем «стоимость индивидуально»
- **Быстрый ответ** — «Отвечу в течение 2 часов» — снимает страх «напишу, а он пропадёт»

---

#### Что заставит закрыть вкладку

- Заголовок про технологии, а не про результат: «Full-stack разработчик, React/Node.js/Python» — ему не важен стек
- Нет ни одного реального проекта с живой ссылкой
- Цен нет совсем и нет даже намёка на диапазон
- Форма с вопросом «Опишите ваш проект» без предварительного контекста — он не знает, с чего начать
- Сайт сам по себе выглядит дёшево или медленно грузится — «если себе такое сделал, мне сделает не лучше»
- Нет мессенджера / Telegram — хочет написать сообщением, а не звонить незнакомому человеку

---

### Общий список рекомендаций — что сделать

#### Структура страницы (порядок блоков)

1. **Герой** — оффер результатом + кому + CTA «Обсудить проект»
2. **Услуги** — 3 направления (сайты / приложения / ИИ-агенты) с кратким описанием и ценой/диапазоном
3. **Кейсы** — 3–5 проектов: задача → решение → результат в цифрах + живая ссылка
4. **О себе** — фото + 3–4 предложения от первого лица + 3–4 цифры (проектов, лет опыта, довольных клиентов)
5. **Отзывы** — 3 отзыва с именем, должностью, компанией, конкретным результатом
6. **Как работаем** — 4 шага от заявки до готового проекта
7. **FAQ** — 5 вопросов закрывающих основные возражения
8. **Финальный CTA** — «Обсудить проект» + Telegram + время ответа

---

#### Оффер — переписать

❌ «Создаю сайты, приложения и ИИ-агентов под ключ»
✅ «Делаю цифровые инструменты, которые работают на ваш бизнес — от лендинга до ИИ-агента»
✅ «Запускаю проекты за фиксированный срок и цену — без исчезновений и бесконечных правок»

---

#### Быстрые победы (сделать в первую очередь)

- [ ] Переписать заголовок первого экрана — результат для клиента, не стек технологий
- [ ] Добавить реальное фото и имя
- [ ] Сделать 2–3 кейса с цифрами и живыми ссылками
- [ ] Указать хотя бы диапазон цен
- [ ] Убрать форму с 5+ полями → оставить имя + Telegram
- [ ] Добавить «Отвечу в течение 2 часов» рядом с CTA
- [ ] Проверить мобильную версию прямо сейчас

---

#### Главная мысль

Лендинг портфолио проигрывает не потому что плохой дизайн, а потому что говорит о себе, а не о клиенте. Клиент приходит с вопросом «ты решишь мою проблему?» — страница должна отвечать на этот вопрос в первые 5 секунд.

## Раздел A. Конкуренты и примеры лендингов

### 1. LingoGalaxy
**lingogalaxy.com**

- **Что цепляет:** космическая тема — тёмно-синий фон, планеты-иконки, анимации и ракеты. Не просто школа, а «галактика языков»
- **Секции:** герой → галактика языков (12 планет) → сегментация по возрасту → отзывы → как работает процесс
- **Стиль:** детский, игровой, тёмный фон + яркие акценты
- **Взять себе:** тёмный фон работает и для детской аудитории; метафора-образ как объединяющий мотив всего лендинга; разделение по возрасту прямо на странице

## Раздел B. Идеальная структура лендинга (8–10 блоков)

> Исследованы: Novakid, Skyeng, Skysmart, GetCourse, WhiteEdtech, vc.ru, smashingmagazine.com

### Блок 1 — Первый экран (Above the Fold)
**Цель:** за 3–5 секунд ответить на вопрос «что это и зачем мне это?»

Внутри:
- Заголовок — конкретный результат, не описание: «Ваш ребёнок заговорит на английском за 4 месяца» (не «онлайн-школа английского»)
- Подзаголовок — уточнение: возраст, формат, без долгих обязательств
- Фото/видео — живой ребёнок на уроке, не стоковая картинка
- Один CTA — «Записаться на бесплатный урок» с контрастной кнопкой
- Социальное доказательство в одну строку: «Уже учатся 120 детей»

### Блок 2 — Проблема и боль родителей
**Цель:** показать, что мы понимаем ситуацию. Родитель должен узнать себя.

Внутри: список из 3–4 узнаваемых ситуаций → фраза-мост: «Мы сделали так, чтобы это больше не было вашей проблемой»

Примеры болей:
- «Репетитор дорогой, а результата нет»
- «Ребёнок не хочет учиться»
- «Скоро ОГЭ, а уровень — ноль»

### Блок 3 — Для кого
**Цель:** помочь родителю идентифицировать себя, отсечь нецелевых.

3–4 портрета: «Ребёнок 10–12 лет, начинает с нуля» / «Подросток 14–17, готовится к ОГЭ/ЕГЭ» / «Хочет смотреть YouTube без субтитров»

### Блок 4 — Что получит ребёнок (результаты) ⭐ Самый важный
**Цель:** продать результат, не процесс.

Конкретные, измеримые результаты: «Научится строить предложения без подсказок», «Сдаст ОГЭ без стресса», «Начнёт понимать песни и сериалы». Никаких «вы получите уникальную методику». Можно «До → После».

### Блок 5 — Как проходят занятия
**Цель:** снять тревогу «а вдруг скучно / непонятно»

3–5 шагов или карточек: живой преподаватель, видеозвонок, домашние задания без зубрёжки, игровые элементы. Короткое видео с фрагментом урока (повышает конверсию до 86%).

### Блок 6 — Преподаватель
**Цель:** создать доверие к конкретному человеку, не к абстрактной «команде»

- Фото — живое, не постановочное
- Имя, образование, стаж, специализация
- 1–2 строки от первого лица: «Я работаю с детьми 7 лет и знаю, как объяснить Present Perfect без слёз»

### Блок 7 — Отзывы и результаты учеников
**Цель:** социальное доказательство. Родитель доверяет другим родителям.

- Отзывы с именем, городом, реальным фото
- Конкретные результаты: «За 3 месяца с A2 до B1», «Сдала на 5»
- Скриншоты переписки из Telegram/WhatsApp — выглядят живо

### Блок 8 — Цены и пробный урок
**Цель:** снизить финансовый барьер входа.

- 2–3 тарифа, один выделить как «популярный»
- Бесплатный пробный урок как главный CTA
- Открытые цены — без них 80% уходят

### Блок 9 — FAQ + финальный CTA
**Цель:** закрыть последние возражения.

Топ-вопросы:
- «А если преподаватель не понравится?»
- «Как записаться?»
- «Можно ли перенести урок?»
- «Какая платформа? Нужно ли что-то скачивать?»

Финальный CTA — тот же, что на первом экране.

### Что обязательно должно быть на первом экране

| Элемент | Почему важен |
|---|---|
| Конкретный результат в заголовке | Понятно, зачем платить |
| Подзаголовок с возрастом и форматом | Быстрая идентификация ЦА |
| Реальное фото ребёнка на уроке | Доверие, не сток |
| Один CTA «Записаться на бесплатный урок» | Единственное действие |
| Одна строка с цифрами | Быстрое соцдоказательство |
| Мобильная версия — большая кнопка | 70% трафика с телефона |

### Топ-5 ошибок, убивающих конверсию

1. **Продают процесс, не результат** — «уникальная методика» не продаёт. Нужно «ребёнок заговорит за 4 месяца»
2. **Фальшивые отзывы** — без имени, города, фото. Родитель всегда ищет «таким же детям помогло?»
3. **Несколько CTA одновременно** — «купить / позвонить / скачать / подписаться» → пользователь теряется и уходит
4. **Нет цены** — «оставьте номер, расскажем» воспринимается как «дорого и стыдно признать»
5. **Плохая мобильная версия** — мелкий текст, кнопки вплотную. 70% трафика с мобильных, тест: откройте прямо сейчас на телефоне

### Чек-лист перед запуском

- [ ] Заголовок первого экрана содержит конкретный результат
- [ ] Есть бесплатный пробный урок как главный CTA
- [ ] Отзывы — с именем, городом, фото, конкретным результатом
- [ ] Цены указаны открыто
- [ ] Форма захвата — максимум 2 поля (имя + телефон/Telegram)
- [ ] Страница открывается на мобильном без горизонтального скролла
- [ ] Есть живое фото преподавателя
- [ ] CTA повторяется минимум 3 раза по странице

### Топ-5 микроанимаций — реализация без библиотек

**1. Кнопка-пульс (Pulse CTA)**
```css
@keyframes pulse {
  0%, 100% { transform: scale(1); box-shadow: 0 0 0 0 rgba(255,107,43,0.4); }
  50%       { transform: scale(1.03); box-shadow: 0 0 0 12px rgba(255,107,43,0); }
}
.btn-primary { animation: pulse 2.5s infinite; }
```

**2. Появление карточек снизу (Fade-in Up)**
```css
.card { opacity: 0; transform: translateY(30px); transition: opacity 0.5s ease, transform 0.5s ease; }
.card.visible { opacity: 1; transform: translateY(0); }
```
+ `IntersectionObserver` в JS для добавления класса `.visible`

**3. Анимированные счётчики цифр**
JS `setInterval` + `IntersectionObserver` — цифра «считается» от 0 до целевого значения, когда блок входит в экран. Сильный психологический эффект.

**4. Card Lift при наведении**
```css
.card { transition: transform 0.25s ease, box-shadow 0.25s ease; }
.card:hover { transform: translateY(-6px); box-shadow: 0 20px 40px rgba(0,0,0,0.3); }
```

**5. Animated Underline на ссылках**
```css
.link::after { content:''; position:absolute; left:0; bottom:-2px;
  width:0; height:2px; background:#FF6B2B; transition: width 0.3s ease; }
.link:hover::after { width: 100%; }
```

### Шрифты в тренде (Google Fonts, бесплатно)

**Для заголовков:**
- **Nunito** (variable) — округлый, дружелюбный, читается на мобильном. Идеален для детской аудитории
- **Raleway** (variable) — элегантный, лёгкий, подходит для H1 на тёмном фоне
- **Space Grotesk** — геометричный, технологичный, современный

**Для текста:**
- **Inter** (variable) — стандарт читабельности

**Правила типографики:**
1. Fluid type: `font-size: clamp(1rem, 2.5vw, 1.25rem)` — плавное масштабирование
2. Разреженный tracking для больших заголовков: `letter-spacing: -0.02em`
3. `line-height: 1.5–1.7` для body — главный критерий читабельности на мобильном
4. Максимум **2 шрифта** на странице

### Приоритеты для ИН.ЯЗ — что даёт быстрый прирост конверсии

1. **Trust-first блоки** — реальные цифры + фото Светланы → просто, быстро, эффективно
2. **Микроанимации** — pulse на кнопке, card-lift, fade-in при скролле → одна функция JS + CSS
3. **Sticky CTA на мобильном** — кнопка «Записаться» всегда видна → 3 строки CSS
4. **Блок «Боли родителей»** — сейчас его нет на лендинге, а он даёт сильный emotional hook
5. **Явные цены** — сейчас цена есть, но можно оформить как тарифную таблицу — это снижает барьер
